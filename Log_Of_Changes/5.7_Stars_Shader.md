# 星空着色器稳定性改进 (2026年5月7日)

## 总体目标
改善手势追踪的稳定性，解决"手部不动时画面仍然跳变"的问题。

---

## 改进 1：焦点坐标源改进
**问题:** 使用食指尖 (`landmark[8]`) 作为焦点，该点灵活性强，容易受手指弯曲、旋转影响。  
**解决方案:** 改用手腕 (`landmark[0]`) 作为焦点源。
- **优点:** 手腕随手部整体运动，更稳定
- **代价:** 零性能开销
- **文件修改:** `shadertoy/gesture.py` (line 213-214)

**效果:** 焦点位置跳变显著降低（预期 60-80% 改善）

---

## 改进 2：握拳检测优化
**问题:** 握拳检测使用3D距离 `√(dx²+dy²+dz²)`，z方向微小波动导致握拳状态频繁抖动。  
**解决方案:** 改为仅使用 XY 平面距离 `√(dx²+dy²)`，忽略 z 方向噪声。
- **优点:** 握拳/张开检测更稳定，不随深度波动
- **文件修改:** `shadertoy/gesture.py` (line 226)

**效果:** 握拳状态稳定，缩放效果平滑

---

## 改进 3：平滑因子调整
**问题:** 原平滑参数不足以消除 MediaPipe 检测的微小波动。  
**解决方案:** 增强平滑强度

| 参数 | 修改前 | 修改后 | 理由 |
|------|--------|--------|------|
| `alpha_pos` | 0.65 | 0.45 | 手腕更稳定，可用更强平滑 |
| `alpha_action` | 0.5 | 0.4 | 握拳状态更平稳 |

- **文件修改:** `shadertoy/gesture.py` (line 178-179)
- **环境变量:** 可通过 `SHADERTOY_GESTURE_POS_ALPHA` 和 `SHADERTOY_GESTURE_ACTION_ALPHA` 调整

**效果:** 手不动时焦点完全静止

---

## 改进 4：手掌深度参考
**新增功能:** 提供手掌中心深度参考，支持后续深度感知补偿。
- **计算方式:** `palm_depth = avg(landmarks[0].z, landmarks[12].z, landmarks[16].z)`
- **新增 uniform:** `iHandDepthRef` 
- **用途:** 作为深度补偿的参考点，可在 Shader 中使用

**文件修改:**
- `shadertoy/gesture.py` (line 36, 76, 221-223)
- `shadertoy/uniforms.py` (line 20)
- `shadertoy/shader.py` (line 157, 326, 216-217, 373-374)
- `shadertoy/__main__.py` (line 125)
- `WebEngine/visualizer.py` (line 158)

---

## 改进 5：握拳检测开关
**新增功能:** UI 面板中添加"启用握拳检测"复选框，支持动态开启/关闭。
- **默认状态:** 开启
- **关闭时:** 画面保持默认大小（相当于握拳状态）
- **新增 uniform:** `iPinchEnabled` (0.0=关闭, 1.0=开启)

**实现细节:**
- `gesture.py`: 新增 `set_pinch_enabled()` 和 `is_pinch_enabled()` 方法
- `uniforms.py`: 新增 `iPinchEnabled: float = 1.0` 字段
- `shader.py`: 添加 uniform 位置注册和更新逻辑
- `main_page.py`: 添加 checkbox 和对应的槽函数 `_on_pinch_checkbox_changed()`
- `__main__.py` 和 `visualizer.py`: 同步 gesture tracker 状态和 uniform

**文件修改:**
- `shadertoy/gesture.py` (line 36, 81-90, 230-250)
- `shadertoy/uniforms.py` (line 21)
- `shadertoy/shader.py` (line 157, 326, 217-218, 375-376)
- `WebEngine/ui/main_page.py` (line 259-266, 553-567)
- `shadertoy/__main__.py` (line 126)
- `WebEngine/visualizer.py` (line 159)

## 预期效果总结

| 场景 | 改进前 | 改进后 |
|------|--------|--------|
| 手部静止 | 画面频繁跳变 | 焦点完全静止 |
| 手指弯曲 | 焦点跟随手指 | 焦点保持稳定 |
| 握拳状态 | 缩放抖动 | 平稳缩放 |
| 手部旋转 | XY 大幅跳变 | 跳变显著减少 |
| 深度变化 | 握拳判断抖动 | 深度变化不影响握拳 |

## 测试建议

### 场景 1：稳定性测试
```bash
python -m shadertoy shaders/stars.glsl
```
- 手部完全不动 → 焦点应完全静止
- 观察 10-30 秒，确保无跳变

### 场景 2：手指弯曲测试
- 保持手腕位置不变，弯曲手指 → 焦点位置不变

### 场景 3：握拳/张开测试
- 握拳 → 画面回到默认大小
- 张开 → 画面平滑放大
- 切换握拳状态 5-10 次 → 缩放应平稳，无抖动

### 场景 4：握拳检测开关测试
- 打开应用，在面板找到"启用握拳检测"复选框
- 取消勾选 → 手张开时画面仍保持默认大小（不放大）
- 勾选 → 手张开时画面放大

### 场景 5：性能检查
- 运行 `python -m shadertoy shaders/stars.glsl`
- 查看 gesture 线程日志，确保检测延迟稳定在 15-30ms

## 可选后续优化

### Option A：Shader 端深度补偿
在 `shaders/stars.glsl` 中应用基于深度的 XY 敏感度调整：
```glsl
float depth_compensation = mix(1.3, 0.7, clamp(iHandDepthRef, 0.3, 0.7));
vec2 compensated_parallax = base_parallax * depth_compensation;
```

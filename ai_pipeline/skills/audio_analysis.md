# Audio Analysis Skill

分析音频数据并输出结构化的元数据报告和风格推荐。

## 铁律

违反以下任何一条，输出无效：
1. `load_audio_from_file` 返回空数组 → 输出「❌ 无法加载音频：[文件名]，原因：[格式不支持/文件不存在/数据为空]」→ **立即停止，不输出任何后续分析或风格推荐**
2. `find_music_by_name` 返回空 → 输出「❌ 未在 MusicLib/ 中找到匹配文件」→ **停止**
3. 工具调用报错 → 如实输出错误原文，不得猜测、幻觉、编造任何数据
4. 音频加载成功后，元数据字段**必须全部包含**，缺一不可

## 流程

### Step 1: 加载音频
- 调用 `load_audio_from_file` 加载音频数据
- 检查返回的 `audio_array` 是否为空
- 失败 → 输出错误原因 → 停止

### Step 2: 提取元数据
调用 `summarize_audio` 获取原始统计特征后，输出以下**全部字段**：
- **采样点数**: X
- **估算时长**: 基于采样点数 / 采样率（默认 44100Hz），输出 X分Y秒 格式
- **峰值**: X.XXXX (0-1 归一化)
- **RMS**: X.XXXX
- **能量特征**: 根据 RMS 和峰值密度推断，输出 低/中/高 + 一句话描述
- **节奏型**: 根据 32 点采样波形推断，输出 平稳/脉冲/渐变/不规则，附带估算 BPM
- **频段分布**: 根据采样点变化速率推断低频/中频/高频的大概占比

### Step 3: 风格推荐
- 调用 `infer_shader_style` 获取风格建议
- 列出 2-3 个推荐风格，每个附带与音频特征的匹配理由（1-2 句）
- 末尾输出：「请确认风格后点击 **确认生成** 开始编写 Shader」
